JSON基礎筆記

認識JSON

JSON全稱JavaScipt Object Notation(JavaScript對象表示法)。JSON 不屬於JavaScript，他只是一種數據格式，其他很多語言也能序列化和解析JSON

JSON 語法

簡單值

簡單值的語法和JS的語法相同，除了undefined。

如 ‘hello world’ 123 等等 都是JSON

但我們一般使用的JSON是一下兩種形式的。

對象

注意點：

  • JSON中的對象要求屬性名都要加上雙引號。

  • JSON對象屬性的值可以是簡單值，也可以是複雜值（嵌套）。

  • JSON對象最後一個屬性值後面沒有逗號，包括內嵌對象。

數組

我添加了三個JSON對象到一個數組，就組成了一個JSON數組
當然裡面還可以一層一層的嵌套，包括嵌套JSON數組。
注意點同上。

JSON 序列化

eval

eval()可以解析JSON，但eval可能會執行JSON里的一些惡意代碼。會有安全問題。所以基本不用他

stringify

stringify 字面上的意思，是字符串化，他可以將JSON序列化為一個JSON字符串

序列化過濾

接上面的json

數組過濾

stringify的第二個參數是一個數組，返回的結果字符串中，只會包含過濾數組中的相應的字符串

函數過濾

如果第二個參數為函數，參數分別是一個屬性名和一個屬性值。

toJSON()

toJSON()方法 会在序列化之前先执行。
toJSON()方法可以作为函数过滤器的补充，将一个对象传入JSON.stringify()后 的序列化顺序：

①若存在 而且 能通过toJSON()获取有效值，将先调用该方法。否则返回对象本身；

②若存在 第2参数，将①返回的值传入 过滤器；

③对②返回的每个值 进行相应的序列化；

④若存在 第3参数，对序列化后的数据 进行相应格式化。
