DOM 编程艺术1

在学习js的过程中，有一本书叫《JavaSctipt DOM 编程艺术》，这是一本很不错的适合js入门的书，这本书我前前后后，仔细的，粗略的，差不多阅读过3、4遍，这里是一些总结。较书本有些地方会深入一些，有些会浅显一些。欢迎给出意见，指正文章内容。

DOM

DOM是文档对象模型（Document Object Model）的简称,它的基本思想是把结构化文档（比如HTML和XML）解析成一系列的节点，再由这些节点组成一个树状结构（DOM Tree）。所有的节点和最终的树状结构，都有规范的对外接口，以达到使用编程语言操作文档的目的（比如增删内容）。所以，DOM可以理解成文档（HTML文档、XML文档和SVG文档）的编程接口。 —-《JavaScript 标准参考教程（alpha）》，by 阮一峰

节点(node)

DOM节点常用到的就是以下元素节点、文本节点和元素节点。

以此为例，分别认识一下。

  • p标签就是元素节点

  • 世界你好 就是文本节点

  • title=”hello world” 就是属性节点

元素节点

  • nodeType = 1
  • nodeName 的值为元素的标签名
  • nodeValue的值为null
  • parentNode 可能是Document 或者Element

文本节点

  • nodeType = 3
  • nodeName = “#text” // text对象
  • nodeValue 为节点所包含的文本
  • parentNode 是一个Element

属性节点

  • nodeType = 2
  • setAttribute、getAttribute、removeAttribute等一些方法

获取元素

使用js来操作Dom，就需要先将dom元素获取过来。用到多的就是以下几个。

getElementById

我将获取过来的元素存在一个叫oDiv的变量之中。
o的意思是指这个存储的内容类型是一个对象(object),
oDiv中的D之所以大写，是用到一种叫驼峰命名的规范。（不知道可以搜索一下）。
getElementById方法是拿到一个对象

getElementsByTagName

变量名aLi的a是指存储内容为array，这样一个命名规范，有利于程序开发。

getElementsByTagName方法是获得的一个数组对象

getElementsByClassName

同样的,getElementsByClassName方法返回的也是一个数组。

遗憾的是getElementsByClassName是在HTML5 DOM 中的方法，有些浏览器还是不支持的。

DOM属性

childNodes

element.childNodes,返回element元素的所有子元素，是一个包含这个元素全部子元素（不包括孙元素）的数组。

开发时经常踩这个坑(文档中的每一样东西都是节点，甚至连空格、换行符都会被解释成节点，全都会包含在childNodes数组里面)，重点标记一下

如果只是想要获得元素节点，可以使用children，虽然不是标准，但是在当前所有浏览器都能工作。可以通过children.length来取得子元素（元素节点）的数量

nodeType

node.nodeType 检查节点的类型
上文也提到了，一般用在if判断里

  • 元素节点的nodeType == 1
  • 文本节点的nodeType == 3
  • 属性节点的nodeType == 2

nodeValue

使用nodeValue 可以改变文本节点的值

firstChild

node.firstChild 等价于 node.childNodes[0]

firstElementChild

类似firstChild,但是代表Element元素。ie8有兼容问题

lastChild

node.lastChild 等价于 node.childNodes(node.childNodes.length-1)

lastElementChild

类似lastChild,但是代表Element元素。ie8有兼容问题

parentNode

node.parentNode 返回node父节点。

nextSibling

node.nextSibling 返回node的跟在node后面的兄弟节点。如果没有后一个的兄弟节点了就返回null。

nextElementSibling

类似nextSibling,但是代表Element元素。ie8有兼容问题

previousSibling

很容易理解，previoursSibling是nextSibling的相反操作。

previousElementSibling

类似previoursSibling,但是代表Element元素。ie8有兼容问题

DOM方法

使用dom方法可以动态创建各种节点，来丰富网页内容。

createElement

使用createElement可以动态的创建一个元素节点.

createTextNode

使用createTextNode创建一个文本节点

appendChild

把新创建的节点插入到某个文档的节点树上最简单的办法，就是让他成为这个文档上的某个节点的子节点

例如，我们把上面创建的文本节点添加成为新创建的元素节点的子节点，再把创建的元素节点添加成为文档的子节点。

把创建的所有节点已经添加到了文档原有的节点上，也就是插入了文档树。
打开浏览器就可以看到

insertBefore

除了appendChild可以将新建节点插入文档，insertBefore(译作在…前面插入，哈哈)也可以。不过insertBefore需要知道三个点。
分别是，父节点，目标节点，新建节点
parentElement.insertBefore(newElement,targetElement);

insertAfter

相应的，有没有一个方法叫insertAfter呢？很遗憾，是没有的，不过我们可以根据parentNode和nextSibling来自己封装一个insertAfter函数。
