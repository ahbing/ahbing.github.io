css居中之绝对定位

本文参考了国内外的文章，整理了css居中的方案。主要介绍一下绝对定位居中，仅供参考。

一个例子

如果想让一个元素水平居中，使用margin:0 auto;但margin:auto;在垂直居中却没有用的？事实上，绝对定位元素只要定义高度，就可以垂直居中。如下：

解释

为什么上面的代码可以做到水平，垂直居中呢？来解释一下。

  1. 在普通文档流中，margin:auto等于给margin-top和margin-bottom设置了0；标准里也提到，如果margin-top和margin-bottom的值为auto，那么他们的值就是0；
  2. position: absolute对象脱离常规流，一个绝对定位的元素，会被带离文档流,所以他在文档流里面是不占空间的。
  3. 设置top: 0; left: 0; bottom: 0; right: 0;可以理解为浏览器会给这个盒子一定的空间，这个盒子能填充满他的父级盒子，可能是body或者position: relative;的容器。
  4. 设置区块的width和height，使其不占据所有可用的空间，而且，浏览器可以在新盒子上计算margin:auto;
  5. 因为区块是绝对定位的，所以他脱离了文档流，右因为设置了margin:auto;浏览器会给margin-top和margin-bottom相同的值，使其在之前的盒子中垂直居中。

容器内居中

使用绝对居中，你可以将你的绝对定位元素块放入相对定位的容器去校准对齐。

在视窗中居中

fixed定位也是脱离文档流，但top，right，bottom，left等属性以窗口为参考点进行定位。所以有些例如登录窗口就可以用这个实现。

偏移

只要margin:auto被定义了，内容区块就会根据你定义的盒子的top left bottom 和 right垂直居中。

你也可以保持内容区块垂直居中，操纵内容区块左移或者右移。使用right: 0; left: auto;去右，使用left: 0; right: auto;去左。

响应

绝对居中最大好处大概是可以用百分比来设置宽高，甚至是可以在响应式的盒子上使用min-width/max-width和min-height/max-height。

顺便一说，绝对居中对padding并不影响。

溢出

当内容的高度超出区块或容器(视窗或者设置了相对定位的元素)的高度将会溢出，而溢出的部分会被容器剪掉。

最简单的方法就是增加overflow:auto,只要内容的区块本身没有比容器高(如果没有给内容区块设置padding，可以设置max-height: 100%来确保)，这将会允许给内容增加一个滚动条。

调节尺寸

增加resize属性就可以调节内容区块的大小了。绝对居中的元素无论你怎么改变尺寸，他仍是居中的，设置min-/max-可以限制内容块的尺寸，阻止他溢出容器。

确保要设置overflow: auto;，这样内容块就可以调节的比内容小了。

注意

  1. 设置max-width/max-height的时候，要考虑区块自身的padding，否则也有可能溢出。
  2. IE8之前不兼容resize

图片居中

这个方法适用图片居中，将图片高度设置为auto.
