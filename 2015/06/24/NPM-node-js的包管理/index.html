NPM--node.js的包管理

npm是nodejs的官方包管理，本文介绍了一些你需要知道的重点可以让你高效的使用npm。

npm包管理是和node.js捆绑安装的，他保存所有nodejs项目的的依赖路径，而且你有权限去管理依赖。

NPM可以很简单复用成千上万个开源的包来解决大量的常见问题，你也可以通过在NPM的注册表，在社区分享你的代码。

因为现在JavaScript可以在客户端和服务器端使用，所有有很多种不同的包：有些是为服务器端准备的，有些实质上是命令行的程序，还有些是为前端开发所创造的，所有的包都在NPM的官方网站上罗列。

本文介绍你可以通过NPM做些什么，也涉及了大量常用的命令。

初始化一个项目

当你使用nodejs启动一个新项目，你需要创建一个package.json文件。这是一个你包依赖路径的清单文件，为你的项目提供元数据，也可以通过命令行执行，有选项的定义一些项目的具体任务。

局部安装

既然你已经有了package.json文件，现在就可以往里面添加依赖罗列你所需要的一切。

去安装在package.json中罗列的包和包的依赖，切换到你项目的路径，执行install命令,就会创建一个node_modules文件夹，把你下载的包全部放在那里。

你也可以在终端记录并安装新的包。

其他一些有用的选择：

  • 从package.js中写在或者删除包：npm uninstall --save <packagename>

  • 直接从git仓库安装包：npm install git://github.com/mrfoo/mypackage.git

  • 安装一个明确版本号的包：npm install somepackage@1.1.0 or npm install somepackage@latest

需要特别注意的是，通过以上方法安装的包是局部的，意味着他们储存在当前的项目，并只为当前项目所用，这种做法适合，让每一个项目定义自己的模块，他们有明确的版本号并且可以单独下载。

管理依赖

NPM可以管理你项目的依赖，和相应的版本。开源的项目往往更新比较快，所以版本号是经常变化的。你可以控制哪些类型更新是可以被package.json接受的。

NPM使用语义化版本,是一个标准来显示，发布实施的代码有哪些变化，因为他会影响代码的稳定性。

发行号通常包含三个元素：

  • 主要版本打破向后兼容性，如2.0.0

  • 次要版本添加新功能，不打破现有功能，如1.1.0

  • 补丁发布包含bug修复或者其他小的改动，如1.0.1

定义包的版本

比方说你现在用一个明确的版本号开发。你可以通过指定接受更新的范围来保持更新并保证兼容性。

想要获得更精细的方法，可以访问NPM语义化文档

确定使用最佳实践和补丁发布版本的包，那么你的代码不会被中断。

注意:如果你不确定使用哪个版本，就设置*，然后执行npm update --save，他将安装模块，并会改变星号安装带有^的数字版本号，他将接受较小的或者补丁更新。

设置NPM脚本

NPM有一个内置的run-script命令(或者用npm run简短一些)，这有点像一个操作系统里的别名。

你在scripts对象中定义个别名作为属性，他就会将属性的值作为命令在操作系统默认的shell中执行。

还有一些可使用的npm的快捷键：npm test, npm start, npm stop 和 npm restart。

如果你执行npm test，这和在系统默认的shell上跑npm run-script test或者npm run test是一样的，即mocha test/。

开始的脚本是被定义了的，比如在node.js中开启主文件app.js，"start": "node app.js"

当我们执行npm run lint(或者其他脚本)，NPM也会用pre-和post-挂钩执行相应的命令，就像npm run prelint,npm run postlint。这就给我们编程一个机会，去控制命令发生前后的一些东西。

通过配置脚本文件，你甚至可以消除grunt和glup等编译工具。你可以查看如何使用NPM就像编译工具的文章

开发依赖

只有在开发时候用到的依赖，可以单独添加到依赖列表里。

devDependencies是意味着只有在开发时用到的包，比如单元测试，linting, minification and similar.在大多数情况下，如果你是一个终端用户，你只需要dependencies

默认情况下，当你在项目目录执行npm install，他会安装dependencies以及devDependencies(因为这里已经假设你在源码目录，所有你一定是一个开发者)。

如果你增加--production标签，他只会安装dependencies。

如果你用npm install <package>从公开的注册表安装，他将只会安装dependencies，因为你只是包的使用者。

在终端安装包并保存在package.json下的devDependencies中，需要增加--save-dev标签。

Stackoverflow上有回答清晰的解释了dependencies和devDependencies之间的不同

总个结

npm让安装，管理，发布包变得十分简单，你可以通过创建package.json文件，在里面列出依赖，并安装，开启一个新项目。命令行工具安装全局软件包。

虽然你的工作是定期检查更新，添加一些新的依赖，编写自定义脚本。你也可以将依赖分成devDependencies和dependencies，是他们在生产环境和开发环境有所区别。

当你完成了项目，可以将它发布，让其他开发者使用。
